# Mihomo VPN на Keenetic — установщик для пользователей

Готовый скрипт, который превращает любой Keenetic с Entware в полноценный VPN-роутер с автообновлением подписки и удобной веб-панелью «Включить / Выключить VPN».

## Что вы получаете

- **Mihomo (Clash.Meta)** на роутере, обновляется автоматически
- **SOCKS5 + HTTP-прокси** на `192.168.1.1:1080` (Mixed-port) для устройств, которые сами хотят VPN
- **Веб-панель** на `http://192.168.1.1:8181/` с двумя большими кнопками — для тех, кто не любит SSH
- **Zashboard** (продвинутый дашборд Mihomo) на `http://192.168.1.1:9090/ui/`
- **Автообновление подписки** каждый час с горячей перезагрузкой (без обрыва соединений)
- **Аварийный kill switch** одной кнопкой / одной командой
- **Безопасно по умолчанию**: TUN отключён, перехвата LAN нет — если mihomo упадёт, обычный интернет в квартире не отвалится

## Что нужно перед запуском

1. **Keenetic** с поддержкой Entware (большинство современных моделей: Giga, Ultra, Hopper и т.п.)
2. **Entware/OPKG** установлены на USB-флешку или диск.
   Если не установлены — зайди в веб-морду Keenetic → «Приложения» → найди и поставь OPKG → перезагрузи роутер.
3. **SSH доступ** root в Entware. По умолчанию `root@192.168.1.1`, пароль `keenetic`.
4. **URL подписки** в формате Mihomo/Clash.Meta. С HWID-защитой (Remnawave) тоже работает — мы передаём правильные заголовки.

## Установка

### Способ 1 — скопировать и запустить

С компьютера:

```sh
scp install.sh root@192.168.1.1:/tmp/
ssh root@192.168.1.1 'sh /tmp/install.sh'
```

### Способ 2 — wget с GitHub-релиза (если выложите)

```sh
ssh root@192.168.1.1
wget -O /tmp/install.sh https://example.com/install.sh
sh /tmp/install.sh
```

### Способ 3 — без интерактива (для опытных)

```sh
SUB_URL='https://your.subscription.example/abc123' \
HWID='keenetic-myrouter-001' \
PANEL_PORT='8181' \
sh /tmp/install.sh <<< 'y'
```

## Что вас спросит скрипт

| Поле | Что туда |
|---|---|
| `URL подписки` | Полный https-URL подписки с твоим токеном |
| `HWID` | Любая уникальная строка. По умолчанию генерится. Запомни — если поменяешь, сервис может посчитать это новым устройством |
| `x-device-os` | По умолчанию `KeeneticOS` — оставь как есть |
| `x-device-model` | Произвольное название роутера |
| `User-Agent` | По умолчанию `clash.meta` — нужен для Remnawave-провайдеров, чтоб они отдали Mihomo-формат |
| `Порт веб-панели` | По умолчанию `8181` |

## После установки

В терминале появится:

```text
Веб-панель  :  http://192.168.1.1:8181/
Дашборд     :  http://192.168.1.1:9090/ui/
SOCKS5/HTTP :  192.168.1.1:1080
```

### На клиентских устройствах

VPN включается явно — в системных настройках прокси на устройстве:

- **macOS**: System Settings → Network → Wi-Fi → Details → Proxies → SOCKS Proxy: `192.168.1.1:1080`
- **iOS**: Wi-Fi → твоя сеть → Configure Proxy → Manual → `192.168.1.1:1080`
- **Android**: Wi-Fi → твоя сеть → Advanced → Proxy → Manual → `192.168.1.1:1080` (нужен HTTP-прокси, наш Mixed-port подходит)
- **Windows**: Proxy в настройках → SOCKS5 `192.168.1.1:1080`
- **Браузеры**: SwitchyOmega / FoxyProxy с профилем SOCKS5 `192.168.1.1:1080`

Mihomo сам разруливает что через VPN, а что напрямую — это в правилах подписки. Российские сайты не тормозят, заблокированные открываются.

## Аварийные сценарии

### 1. Открой веб-панель и нажми «⛔ Выключить VPN»

```text
http://192.168.1.1:8181/
```

После этой кнопки VPN не вернётся даже после ребута роутера, пока ты не нажмёшь «✅ Включить VPN».

### 2. Если веб-панель недоступна — SSH-команда

```sh
ssh root@192.168.1.1 /opt/sbin/mihomo-panic.sh
```

Пароль: `keenetic`.

### 3. Если SSH не отвечает — Telnet в Keenetic CLI

```sh
telnet 192.168.1.1
```

Логин/пароль — от админки Keenetic. Внутри:

```text
system reboot
yes
```

Mihomo после ребута сам не поднимется (если до этого был выключен через панель или panic.sh).

### 4. Полный физический сброс

Просто выдерни питание роутера на 10 секунд. После старта VPN не активируется, обычный интернет вернётся.

## Структура установки

| Файл | Зачем |
|---|---|
| `/opt/sbin/mihomo` | Бинарь mihomo (ставится через XKeen) |
| `/opt/sbin/yq` | YAML-парсер (нужен для нашего скрипта) |
| `/opt/etc/mihomo/config.yaml` | Активный конфиг (пересобирается каждый час) |
| `/opt/etc/mihomo/.disabled` | Флаг kill-switch. Если есть — VPN не запустится |
| `/opt/etc/mihomo/zash/` | Zashboard UI |
| `/opt/sbin/update-mihomo-sub.sh` | Cron-скрипт обновления подписки |
| `/opt/sbin/mihomo-start.sh` | Чистый запуск mihomo |
| `/opt/sbin/mihomo-resume.sh` | Снять kill-switch и поднять |
| `/opt/sbin/mihomo-panic.sh` | Kill switch (паника) |
| `/opt/sbin/mihomo-panel.py` | Web-панель (Python stdlib) |
| `/opt/sbin/mihomo-watchdog.sh` | Cron-сторож: поднимает mihomo если упал (раз в минуту) |
| `/opt/sbin/mihomo-vpn-uninstall.sh` | Полное удаление (xkeen остаётся) |
| `/opt/etc/init.d/S96mihomo-panel` | Автозапуск панели |
| `/opt/etc/init.d/S97mihomo` | Автозапуск mihomo (с уважением kill-switch) |
| `/opt/etc/init.d/S99xkeen` | XKeen, **start_auto="off"** to keep its iptables out of the way |
| `/opt/var/log/mihomo*.log` | Логи |
| `/opt/backups/mihomo/` | Бэкапы конфига (последние 5) |

## Прозрачный режим (TPROXY) — для телика и устройств без настроек прокси

По умолчанию установщик ставит **режим A**: явный SOCKS5/HTTP на `192.168.1.1:1080`. Это безопасно — если mihomo упадёт, никто из LAN не теряет интернет. Но для умных устройств (LG TV, Nvidia Shield, IoT-камера) которые не умеют SOCKS5 — этого недостаточно.

Чтобы перевести нужные устройства в **прозрачный режим** (трафик через VPN автоматически, без настроек на устройстве):

1. На роутере один раз включить XKeen-обвязку:
   ```sh
   ssh root@192.168.1.1
   sed -i 's/start_auto="off"/start_auto="on"/' /opt/etc/init.d/S99xkeen
   /opt/etc/init.d/S97mihomo stop && sed -i 's/^ENABLED=yes/ENABLED=no/' /opt/etc/init.d/S97mihomo
   killall mihomo; /opt/sbin/xkeen -start
   ```

2. В web-морде Keenetic `http://192.168.1.1` → **«Список клиентов»** → клик на нужное устройство → в боковой панели «Политика доступа» выбрать **xkeen** → **Сохранить**.

3. Готово. Устройство теперь прозрачно ходит через VPN по правилам подписки.

Чтобы вернуть устройство в обычный режим — там же выбрать «Политика по умолчанию».

**Важно:** в TPROXY-режиме устройства из политики зависят от mihomo. Watchdog раз в минуту проверяет и поднимает его, но между падением и восстановлением может быть до 60 секунд провала.

## Удаление

```sh
ssh root@192.168.1.1 /opt/sbin/mihomo-vpn-uninstall.sh
```

Удалит все наши файлы, init.d-скрипты и cron. Бинарь mihomo и XKeen останутся на месте — если они тебе не нужны:

```sh
ssh root@192.168.1.1 'xkeen -dm; xkeen -remove'
```

## Дизайн-решения

- **Без TUN** — потому что TUN с auto-route на роутере ломает маршрутизацию LAN, и весь дом теряет интернет
- **Mixed-port вместо tproxy** — потому что в случае краха mihomo клиенты, не настроившие прокси явно, ничего не теряют
- **Hot-reload через `PUT /configs?force=true`** — обновление подписки не разрывает активные SOCKS-соединения
- **`MATCH,DIRECT` catch-all** — VPN включается только для конкретных правил из подписки (Telegram, YouTube, заблокированные РКН и т.д.), а не для всего трафика. Это решение продавца подписки, мы только применяем
- **XKeen активен но `start_auto="off"`** — оставляем на случай если потом захочешь переключиться на transparent-режим через Keenetic policy
