# Mihomo VPN на Keenetic — установщик для пользователей

Один скрипт превращает Keenetic с Entware в **прозрачный VPN-роутер**: указанные тобой устройства (телик, ПК, телефон, приставка) ходят через VPN автоматически, без настроек на самих устройствах. Российские сайты, игры и торренты идут напрямую — по правилам подписки.

## Что вы получаете

- **Mihomo (Clash.Meta)** в TPROXY-режиме через [XKeen](https://github.com/jameszeroX/XKeen) — устройства из политики прозрачно проксятся, остальные ходят как обычно
- **Дашборд Zashboard** на `http://192.168.1.1:9090/ui/` — серверы, ноды, лог-стрим, ping-тест
- **Веб-панель Вкл/Выкл VPN** на `http://192.168.1.1:8181/` — для тех, кто не любит SSH
- **Автообновление подписки** каждый час с горячей перезагрузкой (без обрыва соединений)
- **Watchdog** — поднимает mihomo за <60 сек если упал
- **Аварийный kill switch** одной кнопкой / одной SSH-командой
- **Безопасные дефолты**: TUN отключён, нет тотального перехвата — выпали из политики только те устройства, которые ты сам туда добавил

## Что нужно перед запуском

1. **Keenetic** с поддержкой Entware (Giga, Ultra, Hopper, Sprinter и т.п.)
2. **Entware/OPKG** на USB-флешке. Если нет — в web-морде Keenetic → «Приложения» → поставить OPKG → перезагрузить роутер
3. **SSH доступ** под root в Entware. По умолчанию `root@192.168.1.1`, пароль `keenetic`
4. **URL подписки** в формате Mihomo/Clash.Meta. HWID-защищённые (Remnawave) тоже работают

## Установка

### Способ 1 — скопировать и запустить

С компьютера:

```sh
scp install.sh root@192.168.1.1:/tmp/
ssh root@192.168.1.1 'sh /tmp/install.sh'
```

### Способ 2 — прямо с GitHub

```sh
ssh root@192.168.1.1
wget -O /tmp/install.sh https://raw.githubusercontent.com/enkinvsh/dropweb-xkeen/main/install.sh
sh /tmp/install.sh
```

### Способ 3 — без интерактива

```sh
SUB_URL='https://your.subscription.example/abc123' \
HWID='keenetic-myrouter-001' \
sh /tmp/install.sh <<< 'y'
```

## Что вас спросит скрипт

| Поле | Что туда |
|---|---|
| `URL подписки` | Полный https-URL подписки с твоим токеном |
| `HWID` | Уникальный ID этого устройства. По умолчанию сгенерится случайный. Запомни — если поменяешь, сервис может считать это новым устройством |
| `x-device-os` | `KeeneticOS` — оставь |
| `x-device-model` | Произвольное название роутера |
| `User-Agent` | `clash.meta` — нужен для Remnawave-провайдеров |
| `Порт веб-панели` | `8181` |

## ⚠️ ОБЯЗАТЕЛЬНЫЙ шаг после установки

Скрипт поднимает Mihomo и iptables-обвязку, но **сам по себе** ни одно устройство ещё не идёт через VPN. Нужно явно сказать какие устройства проксировать:

1. Открой `http://192.168.1.1/` (логин/пароль роутера)
2. Меню **«Список клиентов»**
3. Клик на устройство (телик, ПК, телефон)
4. В боковой панели в селекторе **«Политика доступа»** выбери **«xkeen»**
5. **«Сохранить»**

Повтори для каждого устройства, которому нужен VPN. Готово — оно теперь прозрачно ходит через mihomo по правилам подписки.

Чтобы убрать устройство из VPN — там же выбрать «Политика по умолчанию».

Проверить кто сейчас в политике:

```sh
ssh root@192.168.1.1 'ndmc -c "show running-config" | grep "policy Policy0\$"'
```

## Аварийные сценарии

### 1. Веб-панель → «⛔ Выключить VPN»

Один клик — Mihomo гасится, iptables-хук снимается, **устройства из политики возвращаются на прямой WAN**, флаг сохраняется. После ребута VPN сам не поднимется до повторного «✅ Включить».

### 2. SSH — если веб-панель не отвечает

```sh
ssh root@192.168.1.1 /opt/sbin/mihomo-panic.sh
```

Пароль: `keenetic`.

### 3. Telnet в Keenetic CLI — если SSH мёртв

```sh
telnet 192.168.1.1
# admin / пароль от админки Keenetic
> system reboot
> yes
```

После ребута mihomo сам не поднимется (если до этого был выключен через панель/panic).

### 4. Полный сброс

Выдернуть питание роутера на 10 секунд. Опасности нет: VPN сам не активируется после старта если флаг `.disabled` стоит.

**Не зажимай Reset на 10+ секунд** — это factory reset Keenetic, сольются все настройки.

## Что внутри установки

| Файл | Назначение |
|---|---|
| `/opt/sbin/mihomo` | Бинарь mihomo (через XKeen) |
| `/opt/sbin/yq` | YAML-редактор |
| `/opt/etc/mihomo/config.yaml` | Активный конфиг (пересобирается каждый час) |
| `/opt/etc/mihomo/.disabled` | Kill-switch флаг (если есть — VPN не запустится) |
| `/opt/etc/mihomo/zash/` | Zashboard UI |
| `/opt/sbin/update-mihomo-sub.sh` | Cron-скрипт обновления подписки (hot-reload) |
| `/opt/sbin/mihomo-watchdog.sh` | Сторож: поднимает mihomo за <60 сек если упал |
| `/opt/sbin/mihomo-resume.sh` | Снять флаг + запустить через xkeen |
| `/opt/sbin/mihomo-panic.sh` | Kill switch + снятие iptables-хука |
| `/opt/sbin/mihomo-panel.py` | Web-панель (Python stdlib) |
| `/opt/sbin/mihomo-vpn-uninstall.sh` | Полное удаление |
| `/opt/etc/init.d/S96mihomo-panel` | Автозапуск панели при ребуте |
| `/opt/etc/init.d/S99xkeen` | XKeen с `start_auto="on"` — поднимает mihomo и iptables |
| `/opt/var/log/mihomo*.log` | Логи |
| `/opt/backups/mihomo/` | Бэкапы конфига (последние 5) |

## Удаление

```sh
ssh root@192.168.1.1 /opt/sbin/mihomo-vpn-uninstall.sh
```

Удалит наши файлы, init.d-сервисы и cron. XKeen и mihomo-бинарь остаются — если хочешь и их убрать:

```sh
ssh root@192.168.1.1 'xkeen -dm; xkeen -remove'
```

## Дизайн-решения

- **TPROXY вместо TUN** — TUN с auto-route ломает маршрутизацию роутера и всей LAN. TPROXY перехватывает только пакеты от устройств с `connmark 0xffffaaa` (тот, что Keenetic ставит на устройства из своей политики). Безопасно для LAN.
- **Mihomo через XKeen wrapper** — XKeen берёт на себя iptables-хуки и netfilter.d-инсталляцию. Мы только конфиг и обвязку.
- **Hot-reload подписки через `PUT /configs?force=true`** — обновление не разрывает SOCKS-соединения и большую часть TPROXY-сессий
- **Watchdog (cron `* * * * *`)** — если mihomo упал, в течение минуты поднимется. В TPROXY-режиме это критично, иначе устройства политики висят на мёртвом порте
- **`MATCH,DIRECT` catch-all** — VPN включается только для конкретных правил из подписки (Telegram, YouTube, заблокированные РКН, AI). Российские сайты, игры и торренты идут напрямую. Это решение продавца подписки, мы только применяем
